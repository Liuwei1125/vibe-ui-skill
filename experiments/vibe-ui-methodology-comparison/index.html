Vibe UI 方法论真实效果对比

同一个任务：设计一个介绍 Vibe UI skill 的 landing page。四版共享同一份产品信息，只改变生成方法论：不用 skill 的裸模型直出、最开始版、外部资源增强版、再叠加 Vibe Gate 质量门版。

查看 notes.md

v-1 · 裸模型直出

方法：只给自然语言 brief，不使用 Vibe UI skill、DESIGN.md、模板 recipe 或检查器。看点是默认模型审美和常见 AI landing 风险。

v0 · 最开始版

方法：18 个精选风格 + 通用 landing prompt + 基础 check。看点是“能生成漂亮页”，但容易有 AI landing 常见气质。

v1 · 资源增强版

方法：使用 open-design:linear-app 和 template recipe。看点是更明确的设计 token、source provenance 和结构。

v2 · Vibe Gate 版

方法：在资源增强基础上加入 invariants、materials status、anti-pattern 和 P0 自检。看点是可交付性更强，且不把借鉴来源命名成自身能力。
