## v1.0.0 Release Notes

## Package Scope

This release publishes Vibe UI as a local-first `DESIGN.md` workflow skill with three package modes:

- `minimal`: core skill files, registry, CLI, prompts, icon, public docs, and Vibe Gate watchlist.
- `standard`: `minimal` plus upstream attribution, Vibe UI template recipes, and curated source `DESIGN.md` files.
- `offline-full`: `standard` plus the 150-system upstream offline resource bundle in `resource/open-design-systems.json`.

The repository also keeps `resource/awesome-design-md-main` as a local source bundle for curated built-in styles and future expansion. Runtime commands use styles listed in `registry.json` plus the optional bundled upstream source catalog.

`registry.json` exposes a curated v1.0 set of 18 styles:

- `linear`
- `vercel`
- `stripe`
- `apple`
- `cursor`
- `openai`
- `notion`
- `raycast`
- `mintlify`
- `framer`
- `airbnb`
- `shopify`
- `feishu`
- `doubao`
- `xiaohongshu`
- `wechat-reading`
- `slack`
- `figma`

Runtime commands only use styles listed in `registry.json`.

## v0.2 Additions

- Chinese product-inspired style recommendations for domestic/team/workspace goals.
- Generated `DESIGN.generated.md` on `use`, with provenance and brand-safety guardrails.
- `node scripts/design.mjs remix <primary> <secondary> [goal]` style remix prototype.
- Token-aware `check` patch suggestions retained from the post-v0.1 improvement slice.

## v0.3 Additions

- `node scripts/design.mjs like <design_id> [page_type] [--strength light|medium|bold]` lightweight style prompt generation.
- Page-specific recipes for common page types without expanding the validator into a heavy review system.
- Strength control for light, medium, or bold style influence.
- Brand-safe wording for "Stripe-like" or "Linear-like" workflows as inspiration, not pixel-perfect replication.

v0.3 deliberately avoids spending too much effort on complete style reproduction. The goal is a useful prompt kit for visual direction, not a full clone checker.

## v1.0 Additions

- Visual preview site generated by `node scripts/design.mjs preview --out vibe-ui-preview`.
- Online design browser data and static page generated by `node scripts/design.mjs browse --out vibe-ui-browser`.
- User-submitted DESIGN.md storage with `node scripts/design.mjs submit <design_id> <DESIGN.md>`.
- URL-to-DESIGN.md extractor with `node scripts/design.mjs extract-url <url_or_html_file>`.
- Figma/screenshot import with `node scripts/design.mjs import <figma_or_screenshot_notes> --kind figma|screenshot`.
- Design consistency report writing with `node scripts/design.mjs report <file_or_directory> --out DESIGN-REPORT.md`.
- Vibe Gate workflow with `workflow`, `invariants`, `brief-check`, persisted contract, report score/decision/blockers, and Bad/Fix/Evidence guidance.
- Offline upstream source access with `--source open-design` and namespaced ids such as `open-design:linear-app`.
- Vibe UI template recipes distilled from upstream source patterns.
- ClawHub/SkillHub-style publishing kit with `node scripts/publish-kit.mjs --platform all --package minimal|standard|offline-full`.
- Publish preflight checks with `node scripts/publish-kit.mjs --platform all --package <mode> --check`.
- GitHub-ready repository files: `README.md`, `README.zh-CN.md`, `CHANGELOG.md`, `AGENTS.md`, `SECURITY.md`, `LICENSE`, issue templates, PR template, and CI workflow.

These v1.0 features are local-first and Skill-friendly. Extracted and imported DESIGN.md files are drafts; review provenance, rights, and brand-safety before publishing.

## Verification

Run before publishing:

```bash
npm run release:check
npm run release:dry-run
npm run release:smoke
```

Optional smoke test from any temporary project:

```bash
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs use linear
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs preview --out preview
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs browse --out browser
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs like stripe pricing --strength light
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs generate landing
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs remix linear feishu "AI collaboration dashboard"
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs check src/app/page.tsx
node /absolute/path/to/skills/vibe-ui/scripts/design.mjs report src/app/page.tsx --out DESIGN-REPORT.md
node /absolute/path/to/skills/vibe-ui/scripts/publish-kit.mjs --platform all --package minimal --check
```

The checker reports a score, issues, suggested fixes, patch suggestions, and DESIGN.md token candidates when it detects unsupported gradients, arbitrary colors, oversized radius, or heavy shadows.

Build all package zips with:

```bash
npm run release:zip
```

Generated artifacts are written to `dist/publish/` and should not be committed. Attach zip artifacts through GitHub Releases if needed.

## Brand Safety

Included styles are inspired by publicly visible UI patterns. They are not official brand design systems, and this skill is not affiliated with or endorsed by the referenced product/company.

Do not copy logos, trademarks, proprietary assets, or official brand claims from inspiration sources.
